## 引言

虚拟机软件（如 VMware、VirtualBox）提供了多种网络模式，用于控制虚拟机与宿主机、局域网及外部网络之间的通信方式。理解这些网络模式的拓扑结构，有助于根据实际需求合理配置虚拟机网络。

本文介绍三种最常见的虚拟机网络模式：桥接（Bridged）、NAT 和 HostOnly，分析各自的工作原理、拓扑结构与适用场景。

## 桥接网络拓扑

![桥接.excalidraw](/images/桥接.excalidraw.svg)

在桥接网络模式下，由虚拟化软件提供一台名为 VMnet 的虚拟交换机，宿主机的物理网卡与虚拟机的虚拟网卡基于虚拟交换机处于同一网段，可以直接通信。

这种模式下，虚拟机可以直接利用宿主机网络路由到外部网络，实现与外部网络的通信。如图，桥接模式下虚拟机直接和物理网络通信，传输效率高。但其缺点也很明显，虚拟机要占用局域网 IP 地址资源。

## NAT网络拓扑

![NAT网络拓扑.excalidraw](/images/NAT网络拓扑.excalidraw.svg)

网络地址转换（Network Address Translation，NAT）的功能是将不同网段的数据通过 NAT 设备转发到另一个网段中，以实现不同网段相互通信的目的。在 NAT 模式下，虚拟化软件除了提供名为 VMnet1 的虚拟交换机外，还生成了一个虚拟路由器作为 NAT 设备，用于连接虚拟机和宿主机所在的两个不同网段。

本质上通过 VMnet1 虚拟机可以实现虚拟机与宿主机之间的通信，但是由于虚拟机和宿主机属于两个网段，虚拟机为了可以正常访问外部网络，则需要通过虚拟路由器 NAT 实现。

虚拟机的虚拟网卡不需要和宿主机的物理网卡处于同一个网段，这样能够节约宝贵的局域网 IP 地址资源。如上图，宿主机的物理网卡属于 192.168.0.0/24 网段，而虚拟机的虚拟网卡属于 1.0.0.0/24 网段，两个网段通过一个虚拟路由器进行通信。在这种模式下，虚拟机和桥接模式一样可以通过宿主机的物理网络访问互联网。

## HostOnly网络拓扑

![HostOnly.excalidraw](/images/HostOnly.excalidraw.svg)

理解了 NAT 网络再理解 HostOnly 网络就更加简单了。相比于 NAT 网络，HostOnly 网络缺少了虚拟路由器 NAT。这种情况下，虚拟机与宿主机之间通过虚拟交换机可以正常通信，但是虚拟机无法访问外部网络。

在一些虚拟机软件中，例如 VMware Fusion 支持用户自定义 HostOnly 的网络是否通过 NAT 连接到外部网络，在选中该选项的情况下可以为该网络增加虚拟 NAT 路由器，如下图。

## 总结

| 模式     | 与宿主机通信 | 访问外网          | 占用局域网 IP |
| -------- | ------------ | ----------------- | ------------- |
| 桥接     | ✅            | ✅（直接）         | ✅             |
| NAT      | ✅            | ✅（经 NAT 转换）  | ❌             |
| HostOnly | ✅            | ❌                | ❌             |

- **桥接**：虚拟机完全融入局域网，适合需要局域网内其他设备直接访问虚拟机的场景
- **NAT**：节省 IP 资源的同时保留外网访问能力，是最常见的默认模式
- **HostOnly**：完全隔离外网，适合搭建纯内部的测试/实验环境
